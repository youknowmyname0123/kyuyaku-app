# 術前休薬チェッカー

お薬手帳・処方箋・薬の説明書などをスマホのカメラで撮影し、術前に休薬が必要な薬剤を
チェックするための補助ツールです。OCR・薬剤データベースとの照合はすべて
**端末内(ブラウザ内)で処理**され、外部サーバーには送信されません。

> ⚠️ **本ツールは参考情報を提供するチェックリスト補助ツールです。**
> 休薬期間は一般的なガイドライン値であり、薬剤の添付文書・各学会のガイドライン・
> 施設のプロトコル・患者個別のリスク(緊急度、腎機能、出血・血栓リスク等)により
> 異なります。最終的な休薬の判断は必ず医師が行ってください。

---

## 1. ファイル構成

```
preop-med-check/
├── index.html      … 画面構成
├── style.css       … デザイン
├── app.js          … 薬剤データベース・OCR・判定ロジック
├── manifest.json   … PWA設定(ホーム画面追加用)
├── sw.js           … オフライン用サービスワーカー
└── icon.svg        … アイコン
```

OCR(文字認識)には [Tesseract.js](https://github.com/naptha/tesseract.js)(日本語モデル)を
CDN経由で読み込んで使用しています。**Claude本体やAPIキーは不要**で、スマホのブラウザだけで
動作します。

## 2. 動作確認方法

### 簡単な方法:スマホのブラウザで直接開く
`index.html` をスマホに転送し、ブラウザで開くだけでも基本的な動作(写真選択・OCR・判定)は
確認できます。ただしカメラの「capture」属性やPWAのインストール(ホーム画面追加)、
オフラインキャッシュ(サービスワーカー)は **httpsで配信した場合のみ正しく動作**します。

### きちんと試す:Webサーバーとして配信する
化学療法ツールと同じく、GitHub → Vercel でホスティングするのがおすすめです。

1. このフォルダの内容をGitHubリポジトリにアップロードする
2. Vercelで「Add New Project」→ そのリポジトリを選択
3. フレームワーク設定は「Other / 静的サイト」のままでOK(ビルドコマンド不要)
4. デプロイ後に発行されるURLをスマホで開く

これで `https://〇〇.vercel.app` のようなURLになり、
- スマホの「ホーム画面に追加」でアプリのように起動できる
- 一度開けばオフラインでも起動できる(Tesseract.jsの初回読み込みのみネット接続が必要)

という状態になります。

## 3. 使い方

1. (任意)手術予定日を入力する
2. 「写真を撮る・写真を選ぶ」からお薬手帳・処方箋の写真を選択する
3. 自動で文字認識(OCR)が実行され、テキストが表示される
   - 認識結果は編集可能。誤読があれば手で修正してOK
   - 写真が不要であれば、直接テキストボックスに薬剤名を入力してもよい
4. 「この内容で休薬対象薬をチェックする」を押す
5. 休薬が必要な可能性のある薬剤がカードで一覧表示される
   - 手術予定日を入力していれば、各薬剤の「休薬開始日」と残り日数も表示される
   - OCRの誤読の可能性がある「類似一致」も別枠で表示される
6. OCRで見つからなかった薬剤は、下部の「薬剤を手動で追加する」から検索して追加できる

## 4. 薬剤データベースの編集方法

`app.js` の先頭にある `DRUG_DB` 配列を編集します。1件は以下の形式です。

```js
{
  id: "aspirin",                 // 一意のID(英数字)
  generic: "アスピリン",          // 一般名
  brands: ["バイアスピリン", "バファリンA81"], // お薬手帳に載る商品名(複数可)
  category: "antiplatelet",      // antiplatelet / anticoagulant / diabetes / hormone / supplement
  days: 7,                       // 手術何日前から休薬するか(0=当日朝のみ中止)
  note: "出血リスクのため…",       // 補足説明(任意)
  verify: true                   // 施設・ガイドラインにより値が大きく異なる場合はtrue
}
```

- 新しい薬剤を追加する場合は、配列の末尾(または該当カテゴリ内)に同じ形式で追加してください。
- `category` を新設したい場合は、`CATEGORIES` オブジェクトにも色とラベルを追加してください。
- 院内の基準表ができたら、`days` の値や `note` を院内基準に合わせて更新することをおすすめします。
  (現在は一般的なガイドライン値での初期版です)

### 特殊なフィールド

- `noRestriction: true` … 「特別な休薬は通常不要」と表示し、休薬開始日のタイムラインを表示しません。
  (例:作用持続時間が数時間程度で、複数日の休薬が不要な薬剤)
- `noDefinedPeriod: true` … 「明確な休薬期間の定めなし」と表示します。`days` は不要です。
  (例:多くのサプリメント・健康食品)
- `renalTable: { title, rows: [{ ccr, halfLife, standard, high }], note }` … 腎機能(CCr)・出血リスク別の
  中止期間を表で表示します(現在はダビガトラン/プラザキサに設定済み)。同様の表が必要な薬剤があれば
  同じ形式で追加できます。

## 5. OCR(文字認識)の精度について

- お薬手帳の**印字された文字**(薬局が発行するシール等)は比較的認識しやすい一方、
  **手書き文字**は誤読が多くなります。
- 撮影時は、できるだけ明るい場所で、紙を真上から、文字が大きく写るように撮影すると
  精度が上がります。
- アプリ側で簡易的なコントラスト補正を自動で行っていますが、誤読がある場合は
  認識結果のテキストを直接修正するか、「薬剤を手動で追加する」から検索して追加してください。
- 「類似一致」として表示される薬剤は、OCRの読み取り結果が登録薬剤名に近いが完全には
  一致しない場合の候補です。誤検出・見落としの両方の可能性があるため、必ず内容を
  確認してください。
